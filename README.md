# OpenChat

## 我们为什么做 OpenChat

### 行业正在进入下一阶段

**1. 从「会聊天的机器人」到可执行的智能体（Agent）**  

以 [OpenClaw](https://github.com/openclaw/openclaw) 为代表的一类项目，带动了整个行业：AI 不再停留在对话式 Chatbot，而是快速转向 **可连接外部系统、能执行任务** 的 **Agent（智能体）** 形态。与此同时，原本彼此孤立的软件边界被冲淡——用户已经可以在 **Telegram、Discord** 等通讯工具里，通过 AI 去操作本地或云上的计算机与服务器。人机协作的入口，正在从「单一 App」扩散到「对话与频道」。

**2. 模型能力与授权边界同步上移**  

以 Claude、GPT 等新一代模型为例，幻觉显著降低，系统与工具权限在可控前提下被逐步放开：AI 可以驱动终端、生成更可审计的代码，并进一步触达真实世界——从业务系统到摄像头、机器人等物理设备。这些变化共同说明：**AI 正在进入「能持续作用于环境」的下一阶段**，而不仅是内容生成。

### 国内生态里我们看到的结构性问题

**1. 「类 Claw」很多，开放却很少**  

腾讯、阿里、Kimi、MiniMax 等厂商都在推出各自的「Claw 系」能力，但多数 **既不开放协议、也不开放实现**——这与 OpenClaw 最核心的 **开源与可审计** 形成反差；若智能体只能锁在单一厂商体系内，生态很难真正生长。

**2. 超级 App 仍在加固围墙**  

以微信等为代表的产品，仍在各自体系内堆叠能力，用户不得不在 **多个封闭花园** 之间切换。若长期如此，人与 AI 的协作会被 **割裂的产品体验** 绑死，这与「Agent 跨系统行动」的下一阶段 **是相悖的**。

### OpenChat 想做什么

OpenChat 希望成为 **开放、可私有部署** 的通讯与协作底座：让人与 Agent 的对话 **不被单一厂商锁死**，并与 **OpenClaw 等开放生态** 对齐，而不是再造一座孤岛。具体设计与路线图见 `workflow/` 中的调研与设计文档。
