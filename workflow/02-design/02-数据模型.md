# 02 OpenChat 数据模型（设计稿）

**状态**：设计稿，随实现与 OpenAPI 迭代修订。  
**关联**：[01-技术架构.md](./01-技术架构.md)（分层、集成与安全原则）。

---

## 1. 目标与约定

| 项 | 说明 |
|----|------|
| **存储** | 首版以 **SQLite** 为权威存储；表名、列名在迁移脚本中与本文对齐。 |
| **ID** | 主键统一 **`id`**：推荐 **UUID v7** 或 **雪花/时间有序** 字符串，便于分布式演进；若 MVP 用整数自增，文档中注明迁移策略。 |
| **时间** | 统一 **UTC**，列名 `created_at` / `updated_at`（`TIMESTAMP`）。 |
| **软删除** | 用户/会话是否软删在实现阶段定；消息默认 **硬删除或「仅对用户隐藏」** 二选一，避免与合规冲突。 |
| **多租户** | 首版 **单租户**（单部署单社区）；预留 `tenant_id` 可列为后续，避免过早加列。 |

---

## 2. 核心实体概览

```text
User ──┬── Device / Session
       ├── ConversationMember ─── Conversation ─── Message
       └── ApiKey（Bot / 集成）

Message ─── Attachment（文件元数据，大文件走对象存储）
```

## 4. 会话与成员

### 4.1 `conversations`

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | PK | 会话 ID |
| `type` | TEXT | `dm` \| `group` |
| `title` | TEXT NULL | 群名称；DM 可为空或由端上展示规则决定 |
| `avatar_storage_key` | TEXT NULL | 群头像 |
| `created_at` / `updated_at` | TIMESTAMP | |

**DM 唯一性**：可增加唯一约束：在 `(user_a, user_b)` 规范化序后的 pair 上建唯一索引，或使用「固定规则生成 DM 的 `conversation_id`」；实现时二选一写清。

### 4.2 `conversation_members`

| 字段 | 类型 | 说明 |
|------|------|------|
| `conversation_id` | FK | |
| `user_id` | FK | |
| `role` | TEXT | `owner` / `admin` / `member` |
| `joined_at` | TIMESTAMP | |
| PK | | `(conversation_id, user_id)` |

## 5. 消息（核心）

### 5.1 `messages`

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | PK | 消息 ID |
| `conversation_id` | FK | |
| `seq` | INTEGER NOT NULL | **会话内单调递增**，用于拉取游标与排序 |
| `sender_id` | FK → users NULL | **系统消息**可为 NULL |
| `sender_kind` | TEXT NOT NULL | **`user` \| `bot` \| `system`**；Bot/Agent 产出必须为 `bot`，防 OpenClaw 回灌 |
| `msg_type` | TEXT | `text` / `image` / `file` / `audio` / `custom` 等 |
| `body` | TEXT NULL | 文本正文；富结构可用 JSON 列 `body_json`（二选一在实现锁定） |
| `client_msg_id` | TEXT NULL | **客户端幂等键**；同会话内 UNIQUE，防重发 |
| `reply_to_message_id` | FK NULL | 引用回复 |
| `integration_meta` | TEXT NULL | **JSON**：与 OpenClaw/外部相关的 `trace_id`、`stream_id` 等扩展，避免污染正文 |
| `created_at` | TIMESTAMP | 发送时间（服务端确认时间） |

**索引建议**：`(conversation_id, seq)`、`(conversation_id, client_msg_id)`（唯一，含 NULL 处理按 SQLite 规则）。

### 5.2 `message_stream_chunks`（可选，流式 AI）

若 Agent 回复分多段推送，可单独表或复用 `messages` + 子行；MVP 可合并为 **`messages` + `stream_id` + `chunk_index` + `is_final`** 字段（仍属设计选项，实现择一）。

| 字段 | 类型 | 说明 |
|------|------|------|
| `message_id` | FK | 父消息 |
| `chunk_index` | INTEGER | 从 0 递增 |
| `content` | TEXT | 片段 |
| `created_at` | TIMESTAMP | |

## 7. Bot 与 API Key

用于 **Bot / OpenClaw 入站调用** 与自动化集成。

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | PK | |

| `key_hash` | TEXT | 仅存哈希 |

## 8. OpenClaw 集成（配置与事件）

### 8.1 `integration_settings`（单例或每会话）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | PK | |
| `scope` | TEXT | `global` 或 `conversation` |
| `conversation_id` | FK NULL | 按会话开启 AI 时非空 |
| `provider` | TEXT | 固定 `openclaw` |
| `enabled` | BOOLEAN | |
| `openclaw_base_url` | TEXT NULL | Gateway 地址 |
| `webhook_secret` | TEXT NULL | 出站签名密钥，存哈希或 KMS 后续 |
| `updated_at` | TIMESTAMP | |

### 8.2 出站事件 JSON（逻辑模型，非必存表）

向 OpenClaw 发送的 **HTTP 体** 建议包含：

| 字段 | 说明 |
|------|------|
| `event_id` | UUID，幂等与去重 |
| `conversation_id` | |
| `message_id` | 触发消息 |
| `sender_kind` | 必须为 `user` 才触发 Agent（避免 `bot` 回灌） |
| `text` / `payload` | 正文与结构化负载 |
| `occurred_at` | ISO8601 |

入站 Bot 写入消息时 **强制** `sender_kind = bot`，并在 `integration_meta` 中写入 `openclaw` 侧 trace。
