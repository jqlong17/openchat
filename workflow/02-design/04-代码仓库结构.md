**状态**：设计稿；在首次 `cargo` 初始化时对齐。

## 1. 原则

| 项 | 说明 |
|----|------|
| **单仓库** | OpenChat 产品源码集中在 **`/Users/ruska/project/openchat`**（或团队 fork），与 `workflow/` 协作文档同库。 |
| **Rust workspace** | 根目录 **`Cargo.toml`** `[workspace]` 管理多个 crate，便于共享类型与集成测试。 |
| **过程文档不入 `src/`** | `workflow/` 仅 Markdown；运行时逻辑只在 crate 内。 |

## 2. 仓库根目录（建议）

```text
openchat/
├── Cargo.toml                 # workspace 根
├── README.md
├── .env.example
├── .gitignore
├── workflow/                  # 规则、调研、设计、计划（已有）
├── migrations/                # SQLx 迁移（或放在 openchat-db 内，二选一）
└── crates/
    ├── openchat-server/       # 二进制入口：HTTP/WS、配置、启动
    ├── openchat-core/         # 领域模型、用例、与存储接口
    ├── openchat-db/           # SQLx、仓储实现、迁移（若迁移不放根目录）
    ├── openchat-integration/  # OpenClaw Webhook 出站、签名、重试策略（可选独立）
    └── openchat-api-types/    # 对外 JSON DTO、OpenAPI 生成辅助（可选，防循环依赖）
```

**MVP 简化**：可先 **单 crate `openchat-server`** 包含全部代码，待模块边界清晰后再拆 workspace。

---

## 3. Crate 职责（目标形态）

| Crate | 职责 |
|------|------|
| **`openchat-server`** | `main`、路由挂载、中间件、配置加载、TLS（可选）、**组合**各层。 |
| **`openchat-core`** | 用户/会话/消息领域类型、业务规则（幂等、`sender_kind`）、**trait** `MessageRepository` 等。 |
| **`openchat-db`** | SQLite 连接、`sqlx` 实现仓储、**migrations**。 |
| **`openchat-integration`** | 调用 OpenClaw Webhook、解析响应；与 core 通过 trait 交互，避免 core 依赖 HTTP 客户端细节。 |
| **`openchat-api-types`** | 请求/响应结构与 `serde`；可选由 `utoipa`/`poem-openapi` 生成 OpenAPI。 |
